# Codex 新範本執行方式整理

本文整理目前讓 Codex 執行「人造石色差 Lab 數據生成任務」的 4 種方式。

目前正式建議使用：**方式 4：Codex Skill**。

---

## 方式 1：每次貼完整 V3 指令給 Codex

每次有新範本時，直接把完整 V3 規則貼給 Codex，並修改本次範本名稱與路徑。

範例：

```text
請依照以下規則執行人造石色差 Lab 數據生成任務。

版本：
V3

本次參考範本名稱：
0420

本次參考範本路徑：
G:\我的雲端硬碟\2026Codex\人造石色差\data\0420.csv

...
```

### 優點

- 規則最完整
- Codex 不容易漏掉條件
- 適合第一次建立流程或大幅修改規則時使用

### 缺點

- 指令太長
- 每次複製貼上很麻煩
- 容易因手動修改造成前後不一致

### 目前建議

不建議作為日常使用方式。  
除非要建立新版本規則，例如 V4、V5，否則不需要每次貼完整指令。

---

## 方式 2：建立固定指令檔，讓 Codex 讀取

將完整 V3 規則存成固定指令檔：

```text
人造石色差/instructions/V3_生成色差規則.md
```

之後只需要告訴 Codex 去讀這份檔案，再指定本次範本名稱。

範例：

```text
請依照固定指令檔執行人造石色差 Lab 數據生成任務。

指令檔：
人造石色差/instructions/V3_生成色差規則.md

本次範本名稱：
0420
```

### 優點

- 不需要每次貼完整 V3 規則
- 規則集中管理
- 適合重複使用的 SOP

### 缺點

- 仍需要明確告訴 Codex 指令檔位置
- 若之後有多份規則檔，需注意指定正確版本

### 目前建議

可作為備用方案。  
如果 Skill 無法正常觸發，可以改用此方式。

---

## 方式 3：把短指令模板放在 prompts 資料夾

將常用的短指令存放在：

```text
人造石色差/prompts/
```

例如建立：

```text
人造石色差/prompts/執行V3生成色差_短指令.md
```

內容範例：

```text
請依照固定指令檔執行人造石色差 Lab 數據生成任務。

指令檔：
人造石色差/instructions/V3_生成色差規則.md

本次範本名稱：
0418
```

之後使用時，只要打開短指令模板，修改範本名稱，再貼給 Codex。

### 優點

- 比方式 2 更方便
- 不需要每次重新想指令
- 適合保存常用操作語句

### 缺點

- 仍需要手動複製貼上
- 若短指令沒有同步更新，可能與最新 Skill 或規則不一致

### 目前建議

建議保留。  
可作為日常操作的備用短指令，也可給自己快速查看。

---

## 方式 4：打包成 Codex Skill

目前 Skill 放在：

```text
.agents/skills/stone-lab-generator/
```

主要檔案包含：

```text
.agents/skills/stone-lab-generator/SKILL.md
.agents/skills/stone-lab-generator/references/V3_生成色差規則.md
```

之後可以直接叫 Codex 使用 Skill。

### 最穩定寫法

```text
請使用 $stone-lab-generator，也就是人造石色差生成技能。

本次範本名稱：
0420
```

### 日常方便寫法

```text
請使用人造石色差生成技能。

本次範本名稱：
0420
```

### Codex 應自動理解

若本次範本名稱為：

```text
0420
```

則輸入檔案為：

```text
人造石色差/data/0420.csv
```

輸出檔案為：

```text
人造石色差/output/0420_generated_lab_output.xlsx
```

### 優點

- 最符合 AI Agent 工作流
- 不需要每次貼長指令
- Skill 可讀取固定規則
- 可自動執行、驗證、輸出、commit
- 最能節省手動操作

### 缺點

- 第一次需要建立 Skill
- 若 Codex 沒有正確觸發中文技能名稱，可改用 `$stone-lab-generator` 明確呼叫

### 目前建議

日常正式使用建議固定使用此方式。

## 建議日常使用流程

### Step 1：放入新範本

將新範本 CSV 放到：

```text
人造石色差/data/
```

例如：

```text
人造石色差/data/0420.csv
```

### Step 2：對 Codex 下短指令

建議使用：

```text
請使用 $stone-lab-generator，也就是人造石色差生成技能。

本次範本名稱：
0420
```

### Step 3：Codex 自動執行

Codex 應完成：

1. 讀取 Skill
2. 讀取 V3 規則
3. 讀取指定範本 CSV
4. 生成 100 筆 Lab 數據
5. 輸出 Excel
6. 驗證 generated_data、conversion_vectors、boundary_check
7. 檢查 F2 重複容許規則
8. 回報 boundary_check warning
9. 成功後自動 git commit

---

## 4 種方式比較

| 方式 | 使用情境 | 優點 | 缺點 | 建議程度 |
|---|---|---|---|---|
| 方式 1：貼完整 V3 指令 | 建立新版本、大改規則 | 最完整 | 太長、麻煩 | 低 |
| 方式 2：讀固定指令檔 | Skill 失效時備用 | 穩定、集中管理 | 還要指定檔案 | 中 |
| 方式 3：prompts 短指令 | 保存常用提示語 | 方便複製 | 仍需手動貼 | 中高 |
| 方式 4：Codex Skill | 日常正式使用 | 最自動化 | 需先建立 Skill | 最高 |

## 新範本提醒

當有新範本時，除了生成新色差，也可以考慮做批次比較，例如：

```text
0418 vs 0420
```

用來觀察：

- F2 / D65 平均差異
- L / a / b 偏移
- F2→D65 向量變化
- 當批生產顏色是否有漂移

是否執行批次比較，由使用者依當次需求決定。
